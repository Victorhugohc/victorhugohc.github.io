# Patching

## Overview

Patching is the process of updating software, firmware, and operating systems to fix vulnerabilities, improve stability, and maintain secure operation.

In industrial systems, patching must be carefully managed due to the need for continuous operation and compatibility with existing equipment.

## Purpose

The purpose of patching is to:

- Fix known security vulnerabilities  
- Improve system reliability and performance  
- Maintain compatibility with supported configurations  
- Reduce exposure to known exploits  

Unpatched systems are a common entry point for cyber incidents.

## How It Works

Patching involves identifying available updates, evaluating their impact, and applying them to systems in a controlled manner.

The typical process includes:

- Monitoring for updates from vendors  
- Assessing relevance and risk of each update  
- Testing updates in a controlled environment when possible  
- Scheduling installation during planned downtime  
- Verifying system operation after applying updates  

In industrial environments, not all updates can be applied immediately. Each update must be evaluated against operational risk.

## Applications

Patching applies to all software based components, including:

- Operating systems in servers and workstations  
- SCADA and HMI software  
- PLC firmware and embedded devices  
- Network equipment such as switches and firewalls  
- Security systems and monitoring tools  

It is part of both maintenance and cybersecurity practices.

## Key Considerations

- System availability may limit when patches can be applied  
- Compatibility with existing software and hardware must be verified  
- Some legacy systems may not support updates  
- Vendor recommendations should be followed  
- Risk of not patching must be balanced against operational impact  

Patching decisions in industrial systems require coordination between operations and engineering.

## Practical Notes

- Maintain an inventory of all systems and their versions  
- Prioritize critical security updates over minor improvements  
- Avoid applying updates directly to production without validation  
- Document all changes and update history  
- Plan regular maintenance windows for updates  

Patching is not a one time activity. It requires continuous management to keep systems secure and stable.
